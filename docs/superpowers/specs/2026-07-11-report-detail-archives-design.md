# 月报与年报原始明细归档设计

## 目标

缩短月报和年报页面底部“原始明细”的默认展示长度，同时保持月报整月、年报整年的统计分析口径不变。

## 数据口径

- 归档筛选只作用于“原始明细”。
- 月报统计卡片、分析看板、复盘、导出和标签分组报告继续使用完整月份记录。
- 年报统计卡片、分析看板、复盘、导出和标签分组报告继续使用完整年份记录。
- 明细标题中的数量、日期范围和空状态使用归档筛选后的记录。

## 月报原始明细

- 原始明细提供周选择器。
- 查看当前月份时，默认选择当前自然周（周一至周日）。
- 查看历史月份时，默认选择该月第一天所在的自然周。
- 切换上月、下月或本月时，重新计算并选择目标月份的默认周。
- 如果选中的自然周跨越月份边界，只显示该周与当前月闭区间重叠的日期。
- 用户选择的周没有当前月记录时，显示“该周暂无记录”。
- 明细继续按日期分组展示。

## 年报原始明细

- 原始明细提供“按周”和“按月”两种归档方式，默认按周。
- 当前年份按周时默认当前自然周；历史年份按周时默认该年第一天所在的自然周。
- 当前年份按月时默认当前月份；历史年份按月时默认一月。
- 切换去年、明年或今年时，归档方式重置为按周，并选择目标年份的默认周。
- 周跨越年份边界时，只显示该周与当前年份闭区间重叠的日期。
- 月归档只显示所选月份与当前年份重叠的记录。
- 明细按周时按日期分组，按月时仍按日期分组，避免再次展示整月级的大块长列表。
- 无匹配数据时分别显示“该周暂无记录”或“该月暂无记录”。

## 代码边界

- 在前端纯函数模块中实现默认周期计算、周期范围与报告范围求交、明细筛选。
- 月报和年报页面只负责归档状态、控件和传入筛选后的分组数据。
- 使用现有本地日期键 `YYYY-MM-DD` 和周一至周日定义，不引入后端或数据库修改。
- 周选择器采用 ISO 周格式 `YYYY-Www`，月份选择器采用 `YYYY-MM`。

## 边界与错误处理

- 无效或空周期值退回当前报告范围内的默认周期，不能意外显示完整月份或年份的全部明细。
- 选择不属于当前报告范围的周或月份时，通过范围求交得到空结果，不扩展到其他月份或年份。
- 日期计算使用本地日期构造，避免 UTC 转换导致边界偏移。

## 测试与验收

- 单元测试覆盖当前月本周、历史月第一周、跨月裁剪和空周期回退。
- 单元测试覆盖当前年本周、历史年第一周、当前年本月、历史年一月、跨年裁剪及周/月切换。
- 页面接线测试确认整月/整年数据仍用于统计、导出和报告，归档数据只用于原始明细。
- 运行完整 `pnpm run test` 和 `pnpm run build`。
- 浏览器验证月报与年报默认归档、周期切换及数量变化。
- 仅提交本功能文件，保留用户已有的未提交 `.gitignore` 修改，然后同步本地 `main` 与 `origin/main`。
