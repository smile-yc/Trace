# Trace 阶段 4：成长目标与里程碑量化实施计划

## 目标

把“成长”从独立记录升级为可追溯的目标系统：目标承载职业方向，里程碑定义量化标准，日报、项目和成果自动成为进度证据，最终可用于月度汇报、年度总结和职业复盘。

## 产品边界

- 不生成个人成长总分，避免把不同性质的目标粗暴相加。
- 目标分为职业发展、公司培养、年度目标和学习成长，可暂停、完成或归档。
- 每个目标可配置多个里程碑，支持数量、投入、阶段、持续四种量化方式。
- 进度优先从源数据自动计算；人工修正必须填写原因，并保留修正历史。
- 日报的多能力维度默认平均分配，也允许手动填写比例，合计必须为 100%。
- 提醒必须说明原因并能定位到目标或里程碑，不做无法行动的泛化预警。

## 数据设计

1. 新增 `growth_goals`，保存目标范围、状态、日期、关联能力与归档状态。
2. 扩展 `milestones`，增加目标、量化方式、数据来源、日期范围、能力筛选、成果门槛和人工修正字段。
3. 新增 `milestone_stages`，保存阶段型里程碑的步骤及完成状态。
4. 新增 `milestone_corrections`，保存每次人工修正前后的数值、原因与时间。
5. 旧里程碑迁移为阶段型手工里程碑，保留原有当前值，避免历史进度被重算。

## 自动计算口径

- 数量：成果数、重要问题解决数、项目数。
- 投入：工时或工作当量；选择能力时按能力分配比例折算。
- 阶段：已完成阶段数 / 总阶段数。
- 持续：日期范围内存在相关工作记录的活跃月份数。
- 成果门槛：投入达到进度但成果不足时显示提醒，不篡改投入事实。
- 有人工修正时展示修正值，同时保留自动值、修正原因和证据入口。

## 执行顺序与验收

1. 以迁移和数据库测试建立目标、里程碑、阶段、纠偏历史及自动进度。
2. 以组件工具测试建立能力比例的平均分配、手动分配和 100% 校验。
3. 实现成长与目标页面：目标管理、里程碑编辑、进度、证据、提醒和纠偏。
4. 更新年度导出兼容性，确保现有里程碑报告继续工作。
5. 运行全量测试、类型检查和生产构建；仅合并本地 `main`，暂不推送 GitHub。

## 完成标准

- 能创建并维护四类成长目标。
- 四种里程碑均能按约定数据源计算进度并列出证据。
- 能手动修正进度，且原因和历史可查询。
- 日报多能力可平均或手动分配，非法比例不能提交。
- 页面给出到期、偏离、长期无进展及高投入低成果提醒。
- 不出现成长总分，旧数据和现有导出不丢失。
