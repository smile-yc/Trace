本文件保留产品规划和后续任务，不代表所有条目都已经完成。当前代码已经实现的核心闭环包括：

- 记录增删改查与 SQLite 持久化。
- 配置中心、当量标准、分析权重和预警规则。
- 周/月/年数据展板、工作量趋势、占比图、排行和业务能力关联图。
- 成长地图和知识资产库的独立维护页面。

- 日报项目/非项目关系选择、项目快速新建、旧项目名称保守迁移。

- 日报记录直接关联里程碑。
- 数据导入/恢复入口。
- 登录鉴权和公网部署安全。

## 1. 产品定位

本系统用于记录日常工作、生成日报/周报/月报/年报，并逐步升级为“工作台账 + 工作当量统计 + 个人成长复盘 + 年终总结数据源”系统。

系统目标不是单纯写流水账，也不是单纯统计工作量，而是同时回答两个问题：

- 我每周、每月、每年具体做了什么。
- 我的工作重心、能力成长、目标进度和短板在哪里。

系统最终应服务于以下场景：

- 日常快速记录工作内容，长期保存，不依赖浏览器本地缓存。
- 周报查看本周项目、工作内容和工作当量。
- 月报用于个人复盘，总结新技能、重点项目、新任务、新工具、外出交流学习等。
- 年报输出年终总结素材、培养计划复盘材料、年度工作量统计表。
- 支持围绕培养计划、职称、成长目标进行数据化追踪。
- 通过预警机制提醒能力维度断更、目标占比偏差，帮助查漏补缺。

## 2. 背景依据

### 2.1 当前系统基础

当前系统已具备基础工作台账能力：

- React + Vite 前端。
- Express + SQLite 后端。
- 记录增删改查。
- 日报、周报、月报、年报页面。
- Word / PDF / Excel 导出。
- JSON 备份导出。
- SQLite 持久化，默认数据文件为 `backend/data/report.sqlite`。

当前系统已在数据模型中初步加入工作当量相关字段，并已完成基础配置中心、当量标准管理、周/月/年基础数据展板第一版。能力维度、评分权重、成长复盘增强尚未完整产品化。

### 2.2 Demo 带来的新增方向

Demo 展示的重点能力包括：

- 日报录入时选择业务分类、工作类型、产品系统、子任务。
- 记录工作当量。
- 可选关联成长里程碑。
- 可选沉淀知识资产。
- 成长地图展示目标进度。
- 能力雷达展示不同能力方向投入。
- 薄弱维度提醒。
- 知识资产库自动沉淀典型故障、方案模板、复盘报告、提效工具。

这些能力应作为当前系统的升级方向，但需要围绕用户真实使用目标重新整理，而不是直接照搬 Demo。

### 2.3 个人培养目标依据

根据“2026 年光子星培养计划”，系统需要支持以下成长方向：

- 3 年内培养为懂业务、懂技术的新业务解决方案方向专业骨干。
- 在巩固工程技术基础和项目实践经验的同时，补齐业务理解、需求分析、方案编制和售前支撑能力。
- 逐步具备独立承担新业务或集成类项目解决方案设计的能力。
- 参与光伏、能管、培训仿真、虚拟电厂等新业务技术方案支撑。
- 参与部门知识库建设，沉淀项目资料、方案素材、典型案例和常见问题。
- 到 2027 年底前围绕至少 1 个新业务方向形成专题材料或方案素材。
- 2 年内完成中级工程师职称评定。
- 持续做好项目经验、方案素材和提效成果沉淀。
- 作为预备党员，持续加强理论学习和党性锻炼，争取按期转正。

系统应将这些目标拆成可配置里程碑，而不是写死在代码中。

## 3. 用户目标

### 3.1 核心目标

用户希望系统同时支持“工作量统计”和“个人成长复盘”：

- 通过工作当量、记录条数、投入时间综合判断工作重心。
- 通过能力维度判断个人成长是否均衡。
- 通过周/月/年统计发现阶段性成果和短板。
- 通过导出能力为年终总结、培养计划复盘、年度工作量统计提供素材。

### 3.2 周报目标

周报重点回答：

- 本周做了哪些项目。
- 本周具体工作内容是什么。
- 本周工作当量是多少。

周报应以项目和工作内容为主，统计信息为辅，适合快速回顾一周工作。

### 3.3 月报目标

月报偏个人复盘，重点回答：

- 本月掌握了哪些新技能。
- 本月参加了哪些重点项目。
- 本月接收了哪些新任务。
- 本月学习或使用了哪些新工具。
- 本月是否有外出交流、客户交流、学习培训等经历。
- 本月工作重心是否符合成长目标。

月报不应只是记录汇总，而应形成“阶段复盘”视角。

### 3.4 年报目标

年报重点输出：

- 年终总结素材。
- 培养计划复盘材料。
- 年度工作量统计表。
- 年度重点项目清单。
- 年度能力成长变化。
- 年度知识资产沉淀情况。

## 4. 核心概念

### 4.1 工作记录

工作记录是系统最基础的数据单元。每条记录对应一次工作事项，可以是完整任务，也可以是阶段性工作。

记录字段建议：

```text
id
date
title
content
businessCategory       # 业务分类
workType               # 工作类型
abilityDimension       # 能力维度
projectId              # 稳定项目关系，非项目或历史未关联为空
projectRelation        # project / non_project / unassigned
projectName            # 写入时项目名称快照，改名和合并不回写
productSystem          # 产品系统
subtask                # 子任务
quantity               # 数量
coefficient            # 折算系数
workload               # 工作当量
timeHours              # 投入时间，单位小时
milestoneId            # 可选关联里程碑
knowledgeAssetType     # 可选知识资产类型
knowledgeAssetTitle    # 可选知识资产标题
tags
createTime
updateTime
```

说明：

- 每条日报不强制关联成长目标或里程碑。
- 重要工作可以主动关联里程碑。
- 普通工作只作为台账记录和统计数据来源。
- 新记录必须明确选择项目事项或非项目事项，不能以未关联状态提交。
- 历史记录无法按精确名称安全匹配时保留为 `unassigned`，编辑后必须解决关系。

#### 4.1.1 项目

项目是连接工作投入、工作当量、当前重点、阶段进展和成果产出的主实体。

- 名称用于展示，简称和别名用于搜索；重名、别名冲突需要阻止保存。
- 状态包括计划中、进行中、已暂停、已完成和已归档。
- 项目可记录个人角色、周期、目标、说明和完成总结。
- 归档只停止默认选择，不删除项目或历史记录；归档项目仍可搜索和恢复。
- 合并把来源项目的记录关系和别名迁移到目标项目，来源项目保留并指向目标项目。
- 项目改名和合并都不能修改工作记录中的 `projectName` 历史快照。

### 4.2 工作当量

工作当量用于衡量不同类型工作之间的相对工作量。

默认算法：

```text
工作当量 = 数量 × 折算系数
```

要求：

- 系统可根据当量标准自动带出折算系数。
- 用户可手动修改折算系数。
- 用户修改数量或系数后，系统自动重新计算工作当量。
- 工作当量可作为统计、排行、导出和年终汇总的核心指标。

### 4.3 投入时间

投入时间用于补充工作当量无法完全表达的工作投入。

要求：

- 日报支持填写投入时间，单位为小时。
- 投入时间参与工作重心评分。
- 没有填写投入时间的历史记录不影响展示，可按 0 或空值处理。

### 4.4 工作重心评分

工作重心评分用于判断某个周期内主要精力投向哪里。

默认评分口径：

```text
工作重心评分 =
  工作当量占比 × 50%
  + 投入时间占比 × 30%
  + 记录条数占比 × 20%
```

说明：

- 评分对象可以是项目、业务分类、工作类型、能力维度、产品系统。
- 占比在当前统计周期内计算，例如本周、本月、本年。
- 权重应支持配置，默认值为当量 50%、时间 30%、条数 20%。
- 展示时同时显示原始数据和综合评分，避免只看分数失真。

### 4.5 能力维度

能力维度用于观察个人成长方向和短板。

初始能力维度：

```text
新业务探索
工程技术
售前支撑
项目管理与推进
AI与提效工具
客户交流
```

要求：

- 能力维度应支持配置。
- 日报记录可以选择一个主要能力维度。
- 周/月/年报可以按能力维度统计记录条数、投入时间、工作当量、工作重心评分。
- 能力维度用于查漏补缺预警。

### 4.6 里程碑

里程碑用于追踪成长目标、培养目标、职称目标或阶段性任务。

要求：

- 里程碑必须可自定义配置。
- 支持新增、编辑、禁用、排序。
- 支持设置目标类型、截止日期、目标值、当前进度。
- 日报记录可选关联里程碑，但不强制。
- 禁用里程碑不影响历史记录展示。

里程碑字段建议：

```text
id
name
description
category              # 职称 / 新业务 / 政治进步 / 能力提升 / 其他
targetType            # workload / count / manual / checklist
targetValue
currentValue
deadline
enabled
sortOrder
createTime
updateTime
```

### 4.7 知识资产

知识资产用于把工作经验沉淀为可复用材料。

建议采用“轻量沉淀 + 后续完善”的方式，而不是一开始做复杂知识库。

知识资产类型建议：

```text
典型故障库
方案模板
复盘报告
提效工具/脚本
学习笔记
客户交流纪要
项目材料
其他
```

知识资产字段建议：

```text
id
type
title
summary
sourceRecordId
projectName
productSystem
tags
status                # 草稿 / 可复用 / 已归档
link
remark
createTime
updateTime
```

日报中的知识资产沉淀建议：

- 默认只需要选择类型、填写标题和一句话摘要。
- 系统自动关联来源日报。
- 后续统一在成果管理中补充价值、贡献、汇报表述和证据关系，不保存链接或附件。

## 5. 页面与功能需求

### 5.1 日报页面

日报页面用于快速记录当天工作。

必须支持：

- 默认显示今天。
- 支持前一天、今天、后一天切换。
- 支持日期选择器查看历史日报。
- 支持新增、编辑、删除记录。
- 支持填写标题、内容、标签。
- 支持选择业务分类、工作类型、能力维度、产品系统、子任务。
- 支持在项目事项和非项目事项之间明确选择。
- 项目事项通过名称、简称或别名搜索项目，并支持快速新建。
- 支持填写数量、折算系数、投入时间。
- 自动计算工作当量。
- 可选关联里程碑。
- 可选沉淀知识资产。

交互要求：

- 折算系数可由系统匹配，也可手动修改。
- 数量和系数变化时自动更新工作当量。
- 旧记录编辑时应保留新增字段。
- 未填写新增字段的历史记录仍可正常展示。
- 历史未关联记录在编辑时必须选择项目或改为非项目事项后才能保存。

#### 5.1.1 项目管理页面

项目管理页面用于判断当前工作重点、项目投入和下一步行动。

- 默认展示计划中、进行中和已暂停项目，支持按关键词和状态筛选。
- 表格展示项目、状态、个人角色、工时、原始工作当量、最近活跃和当前重点。
- 详情展示记录数、活跃天数、工时、原始工作当量、业务/产品/能力分布和工作时间线。
- 支持创建、编辑、归档、恢复、合并预览和确认合并。
- 已完成和已归档项目不出现在默认日报候选中，但可通过搜索明确找到。
- 当前阶段成果区显示领域空状态；项目与成果实体的直接关联属于阶段 2。

### 5.2 周报页面

周报页面用于查看一周工作情况。

必须支持：

- 上一周、本周、下一周切换。
- 展示本周项目清单。
- 展示本周工作内容摘要。
- 展示本周工作当量合计。
- 展示本周记录条数、投入时间合计。
- 按项目汇总当量、条数、时间。
- 展示本周原始记录明细。

建议支持：

- 本周能力维度分布。
- 本周工作重心 Top 5。
- 本周知识资产沉淀数量。

### 5.3 月报页面

月报页面用于个人阶段复盘。

必须支持：

- 上月、本月、下月切换。
- 展示本月工作当量、记录条数、投入时间。
- 展示本月重点项目。
- 展示本月新任务。
- 展示本月掌握的新技能。
- 展示本月学习或使用的新工具。
- 展示本月外出交流、客户交流、学习培训等记录。
- 展示本月能力维度分布和工作重心评分。
- 展示本月查漏补缺提醒。

建议支持：

- 月度复盘文本自动生成。
- 用户可手动补充月度总结。
- 月报导出 Word / PDF / Excel。

### 5.4 年报页面

年报页面用于年度复盘和材料导出。

必须支持：

- 去年、今年、明年切换。
- 年度工作当量合计。
- 年度记录条数和投入时间。
- 年度项目排行。
- 年度业务分类统计。
- 年度工作类型统计。
- 年度能力维度统计。
- 年度工作重心变化趋势。
- 年度知识资产沉淀统计。
- 年度里程碑完成情况。

导出目标：

- 年终总结素材。
- 培养计划复盘材料。
- 年度工作量统计表。

### 5.5 成长地图页面

成长地图用于展示自定义里程碑。

必须支持：

- 新增、编辑、禁用里程碑。
- 设置里程碑目标类型、目标值、截止日期。
- 查看里程碑关联记录。
- 查看里程碑当前进度。
- 查看最近一次相关工作。

建议初始内置示例，但用户可删除或修改：

- 中级工程师职称。
- 新业务专题材料。
- 预备党员转正。
- 解决方案能力提升。

### 5.6 数据展板页面

数据展板用于跨周期查看工作重心和能力结构。

必须支持：

- 周、月、年时间范围切换。
- 按项目统计。
- 按业务分类统计。
- 按工作类型统计。
- 按能力维度统计。
- 按产品系统统计。
- 工作当量趋势。
- 投入时间趋势。
- 记录条数趋势。
- 工作重心评分排行。

当前已完成：

- 周报、月报、年报已接入基础数据展板。
- 已支持项目统计、业务分类统计、工作类型统计、产品系统统计。
- 已支持记录条数趋势和工作当量趋势。
- 已支持能力维度统计、投入时间趋势和工作重心评分排行第一版。
- 已支持项目工作量排行和项目汇总卡片。
- 原始明细保留在展板下方。

尚未完成：

- 工作重心评分权重配置。
- 查漏补缺提醒。

### 5.7 成果管理页面

成果管理用于维护正式交付成果、重要问题解决、阶段性进展和可复用资产。

必须支持：

- 按年份、项目、类型、状态、能力和关键词筛选。
- 关联项目、多条来源日报、多个能力和多个里程碑。
- 维护背景目标、完成内容、价值影响、个人角色、具体贡献和汇报表述。
- 跟踪计划中、推进中、阶段成果和已完成的状态轨迹。
- 自动汇总关联日报的工时和原始工作当量。
- 独立归档和恢复，不保存附件或外部链接。

### 5.8 配置中心

配置中心用于管理系统口径。

必须支持：

- 业务分类配置。
- 工作类型配置。
- 产品系统配置。
- 子任务配置。
- 能力维度配置。
- 工作重心评分权重配置。
- 里程碑配置。
- 当量标准配置。

配置项要求：

- 支持新增、修改、禁用、排序、默认值。

### 6.6 projects 与 project_aliases

## 7. 统计与预警规则

### 7.1 周期统计

周、月、年统计均应支持：

- 记录条数。
- 工作当量合计。
- 投入时间合计。
- 项目分布。
- 业务分类分布。
- 工作类型分布。
- 能力维度分布。
- 产品系统分布。
- 知识资产数量。

### 7.2 工作重心统计

工作重心可以按以下维度计算：

- 项目。
- 业务分类。
- 工作类型。
- 能力维度。
- 产品系统。

每个维度展示：

- 工作当量。
- 投入时间。
- 记录条数。
- 综合评分。
- 占比。

### 7.3 查漏补缺预警

默认预警规则：

- 某能力维度连续 30 天无记录时提醒。
- 某能力维度年度目标占比与实际占比偏差超过 10% 时提醒。

建议扩展规则：

- 里程碑距离截止日期不足 30 天且完成率低于 80% 时提醒。
- 本月没有知识资产沉淀时提醒。
- 本月没有新技能、新工具、客户交流或外出学习记录时提醒。

预警应以建议形式展示，不应阻断用户使用。

- 成长地图和知识资产库可以在基础统计稳定后逐步增强。
- 年报和导出依赖全年数据结构，适合在前面功能稳定后完善。
