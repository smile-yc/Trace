# Trace 当前功能清单

> 版本口径：本文基于当前代码实现整理，用于说明系统现在能够完成什么，不包含尚未落地的规划项。

## 1. 产品定位与功能边界

Trace 是一套个人工作事实记录、工作量分析、成果沉淀、成长复盘和周期汇报系统。当前功能围绕一条可追溯链路展开：

`日报事实 -> 项目投入 -> 成果证据 -> 成长目标 -> 周/月/年复盘 -> 汇报与导出`

当前边界如下：

- 面向个人长期使用，不是多人协同项目管理或公司绩效考核系统。
- 以用户录入的工作事实和明确关联为依据，不自动判断工作价值，不生成个人成长总分。
- 成果保存结构化文字和来源关系，不保存附件或外部链接。
- 系统保存原始工作当量，不自动执行“传统业务 85 折”等业务折扣。
- 年度汇报折算只影响本次预览与导出，不回写日报、项目、成果或历史统计。

## 2. 全局导航与通用交互

### 2.1 导航结构

系统当前包含七个顶层板块、九个页面入口：

| 顶层板块 | 页面入口 |
|---|---|
| 今日工作台 | 今日工作台 |
| 工作台账 | 工作台账 |
| 项目管理 | 项目管理 |
| 成果管理 | 成果管理 |
| 成长与目标 | 成长与目标 |
| 复盘与汇报 | 周报、月报、年报 |
| 配置与数据 | 配置与数据 |

### 2.2 通用能力

- 桌面端侧边导航与移动端顶部导航。
- 页面切换时保存各页面滚动位置。
- 全局消息提示，用于反馈保存、导出、导入和接口错误。
- 记录编辑弹窗及删除影响确认。
- 项目、配置项等长选项支持搜索选择。
- 空数据、加载失败、表单校验和危险操作确认状态。

## 3. 今日工作台

**入口：** `记录 -> 今日工作台`

**模块目标：** 快速录入每日工作事实，并承接后续项目、成果、能力与报告统计。

### 3.1 日期与概览

- 默认查看当天，支持选择日期、前一天、后一天和返回今天。
- 展示今日记录数、本月记录数、总记录数和标签总数。
- 按所选日期展示当日记录列表。

### 3.2 日报录入

- 录入标题和日期。
- 选择业务分类、工作类型、产品系统和工作细项。
- 长选项支持输入搜索；允许把临时自定义值保存为配置项。
- 选择项目事项、非项目事项或项目未归属。
- 支持在日报表单内快速创建项目。
- 录入数量、投入时间、二级标签和详细内容。
- 根据工作类型、产品系统、工作细项等条件匹配工作当量标准。
- 标准匹配成功时自动填入折算系数；无匹配或用户修改时允许手动填写。
- 自动计算 `原始工作当量 = 数量 × 实际折算系数`。
- 保存系数来源、匹配标准和标准版本，保证后续可追溯。

### 3.3 能力投入

- 一条日报可关联一个或多个能力维度。
- 多能力默认平均分配，也可手动填写各能力占比。
- 保存前校验能力分配合计为 100%。
- 能力分析按分配比例拆分工时和原始工作当量，不重复累计日报总值。

### 3.4 草稿、复制与记录操作

- 手动保存当前日报草稿。
- 手动清除草稿；不会采用自动保存覆盖用户输入。
- 可把已有日报复制为今天的新记录，复制后仍需确认并提交。
- 可编辑、删除已有记录。
- 删除前检查记录对项目、成果、目标和报告统计的影响。
- 可从单条日报直接发起成果沉淀，并自动带入来源记录和项目。

## 4. 工作台账

**入口：** `记录 -> 工作台账`

**模块目标：** 检索全部工作事实、核对统计口径、发现数据缺口并批量形成汇报或成果。

### 4.1 范围与组合筛选

- 默认查看当前周。
- 支持指定周、指定月、指定年和自定义日期范围。
- 支持关键词搜索标题、内容、项目和标签。
- 支持按项目、项目关系、业务分类、工作类型、产品、工作细项、能力维度、系数来源、成果状态和标签筛选。
- 项目筛选支持名称、简称和别名搜索。
- 支持一键清除组合筛选。

### 4.2 汇总指标

- 当前筛选范围的记录数。
- 投入工时和原始工作当量。
- 有实际项目关联的项目数量。
- 与当前记录有关联的成果数量。

### 4.3 数据质量检查

- 检查缺少项目归属、能力维度、工时、折算系数和详细内容的记录。
- 按问题类型展示数量，并可直接筛选对应问题记录。
- 数据质量提示只帮助补录，不自动改写记录。

### 4.4 记录列表与批量动作

- 记录按日期倒序展示。
- 支持逐条选择或选择当前筛选范围。
- 可对单条记录进行编辑、删除和成果沉淀。
- 可对已选记录或当前筛选范围生成标签分组报告。
- 可把多条已选记录提炼为一个成果，并传递来源记录关系。
- 可导出全部工作记录的 JSON 备份。
- 可在二次确认后清空全部记录，并明确提示其统计影响。

## 5. 项目管理

**入口：** `工作 -> 项目管理`

**模块目标：** 把分散日报聚合为项目投入、轨迹、成果和个人贡献视图。

### 5.1 项目档案

- 新建和编辑项目。
- 维护项目名称、简称、状态、起止日期、个人角色、项目目标和项目说明。
- 搜索项目名称、简称或别名。
- 按状态筛选，并可选择是否查看归档项目。
- 项目名称、简称和系统保留的历史别名用于日报项目搜索及历史名称兼容。

### 5.2 项目汇总与追溯

- 展示关联日报数量、投入工时和原始工作当量。
- 展示项目起止时间与当前状态。
- 汇总业务分类、产品系统和能力投入分布。
- 展示当前重点。
- 展示关联成果，可从项目详情直接新增成果。
- 按日期展示项目工作时间线，并保留日报来源。

### 5.3 项目结项闭环

- 项目结项时填写结项总结。
- 记录目标完成情况、关键成果、个人贡献和后续安排。
- 结项后保留投入、时间线和成果关系，不删除历史数据。
- 已结项项目可重新激活。

### 5.4 项目归档与合并

- 支持归档和恢复项目。
- 支持项目合并预览，先展示将迁移的日报和成果数量。
- 合并后把来源项目关系迁移到目标项目，并保留来源项目的合并状态，避免直接破坏历史关联。

## 6. 成果管理

**入口：** `工作 -> 成果管理`

**模块目标：** 管理可用于汇报、复盘和职业证明的结构化成果。

### 6.1 成果类型与状态

- 支持正式交付、重要问题解决、阶段性进展和可复用资产四类成果。
- 支持计划中、推进中、阶段成果和已完成等状态。
- 自动记录成果状态变化轨迹及变更说明。

### 6.2 成果内容

- 新建和编辑成果标题、类型、状态及时间信息。
- 关联项目，也可记录非项目成果。
- 记录背景与目标、已完成工作或解决的问题、价值影响、个人角色、个人贡献、汇报表述、备注和标签。
- 不保存附件和外部链接。

### 6.3 成果证据关系

- 关联一条或多条来源日报。
- 关联能力维度和成长里程碑。
- 从日报、工作台账或项目进入成果创建时自动带入已有上下文。
- 汇总成果来源日报数量、投入工时和原始工作当量。
- 同一日报可支撑多个成果；跨成果汇总按日报 ID 去重，避免重复计算投入。
- 在成果卡片中查看日报和里程碑证据来源。

### 6.4 检索与生命周期

- 按关键词、成果类型、状态、项目和归档状态筛选。
- 展示成果摘要、项目、状态、来源数量、工时、当量和价值影响。
- 支持归档与恢复，不通过直接删除破坏报告证据。

## 7. 成长与目标

**入口：** `成长 -> 成长与目标`

**模块目标：** 用真实工作、项目和成果证据追踪职业方向、培养计划和年度目标。

### 7.1 成长目标

- 新建职业方向、培养计划和年度目标等不同范围的目标。
- 维护目标名称、描述、截止日期和状态。
- 支持目标完成、暂停、恢复和归档。
- 展示进行中目标、量化里程碑、待处理提醒和进度证据数量。

### 7.2 量化里程碑

- 数量型：成果数量、重要问题解决数量、参与项目数量。
- 投入型：投入工时、原始工作当量，可限定能力维度并要求至少形成一定数量成果。
- 阶段型：用户维护阶段清单并逐项勾选完成。
- 持续型：持续活跃月份数。
- 可设置开始日期、截止日期和目标值。
- 自动计算当前值、完成百分比和匹配证据。

### 7.3 证据与人工纠偏

- 里程碑可展示支撑进度的日报、成果或项目证据。
- 阶段型里程碑支持逐项切换完成状态。
- 自动值不合适时可人工纠偏，但必须填写纠偏原因。
- 保存自动计算值、人工覆盖值和纠偏记录，便于追溯。
- 支持暂停或恢复单个里程碑的跟踪。

### 7.4 行动提醒

- 根据目标截止时间、进度状态和能力目标偏差生成提醒。
- 点击目标提醒可回到对应目标。
- 提醒仅用于个人复盘，不等同于绩效评价。

## 8. 复盘与汇报公共能力

**适用页面：** 周报、月报、年报。

### 8.1 周期导航与基础统计

- 支持前后周期切换和返回当前周期。
- 展示记录数量、活跃天/月、原始工作当量和标签数量。
- 周、月、年统计始终基于各自完整周期数据。

### 8.2 数据展板

- 业务分类占比。
- 能力维度投入占比，按能力分配比例统计。
- 工作类型画像。
- 业务与能力关联矩阵。
- 工时与原始工作当量趋势。
- 项目工作量排行。
- 可配置权重的工作重心排行。
- 产品系统分布。
- 项目汇总及本期自动观察。
- 指标、图表项和排行可打开对应来源日报列表。
- 来源列表按记录 ID 去重，并显示日期、标题、项目、工时和原始工作当量。
- 工作重心分数只表示投入集中程度，不表示工作价值。

### 8.3 成果与周期证据

- 按周期展示正式交付、重要问题解决、阶段性进展和可复用资产。
- 周期成果按成果日期落入当前范围。
- 成果投入仅来自明确关联的日报。

### 8.4 复盘与行动工作区

- “数据判断”查看本期与上期的记录、工时、当量等环比。
- 展示项目投入集中度和成果投入产出证据。
- 根据当前数据生成下一步提醒。
- “汇报内容”生成基于源记录和成果的自动摘要。
- 手工填写成绩与贡献、不足、原因、改进措施、成长与能力、下一周期规划。
- 手工内容支持保存草稿、确认定稿和清空。
- 清空手工复盘不会影响自动统计和源数据。

### 8.5 汇报导出

- 支持导出 Word、PDF 和 Excel。
- 可导出当前周期全部内容。
- 可按项目筛选后导出。
- 可按业务分类筛选后导出。
- 导出内容包含周期概览、业务/工作类型/能力/项目/产品等汇总和工作明细。
- 能力汇总按分配比例计算，不重复累计总工时和总当量。

## 9. 周报

**入口：** `复盘与汇报 -> 周报`

- 按自然周查看本周数据，并支持上周、本周、下周切换。
- 展示本周记录数、有记录天数、工作当量和标签数。
- 展示周度数据展板、成果与进展、复盘工作区和导出面板。
- 支持生成本周标签分组报告。
- 原始明细按日期完整展示本周记录。

## 10. 月报

**入口：** `复盘与汇报 -> 月报`

- 按自然月查看数据，并支持上月、本月、下月切换。
- 展示本月记录数、有记录天数、工作当量和标签数。
- 趋势按月内周次组织。
- 展示本月成果、重要问题和阶段进展。
- 生成月报复盘摘要，并展示能力目标偏差等预警。
- 展示本月与上月对比，支持手工复盘草稿和定稿。
- 支持当前周期、项目或业务分类范围的 Word、PDF、Excel 导出。
- 月报总体统计使用整月数据；原始明细可按周切换查看，避免长列表淹没重点。

## 11. 年报

**入口：** `复盘与汇报 -> 年报`

- 按自然年查看数据，并支持去年、今年、明年切换。
- 展示全年记录数、有记录月份数、原始工作当量和标签数。
- 趋势按月份组织。
- 展示年度代表性成果与进展。
- 展示全年与上一年度的对比、投入集中度、投入产出证据和手工年度复盘。

### 11.1 年度成果包

- 汇总年度项目数量、原始工作当量、投入小时和成果来源日报数量。
- 按项目列出投入当量、工时和成果数量，形成项目贡献材料。
- 按四类成果展示年度成果结构。
- 检查成果来源、价值影响、个人贡献和汇报表述是否缺失，并给出待补充提醒。
- 成果关联投入仅统计本年度来源日报，并按日报 ID 去重。
- 系统不自动评价成果价值。

### 11.2 年度汇报折算

- 默认以 100% 展示年度原始工作当量。
- 用户可为本次年度汇报输入折算比例并预览折算结果。
- 折算仅传入本次年度导出，不修改任何原始数据。

### 11.3 年度明细

- 年报总体分析始终使用全年数据。
- 原始日报明细可按周或按月切换查看。

## 12. 配置与数据

**入口：** `系统 -> 配置与数据`

### 12.1 基础配置

- 维护业务分类、工作类型、能力维度、产品系统和工作细项。
- 支持新增、改名、调整顺序、设为默认、启用、禁用和删除。
- 系统项和默认项有明确标识。
- 删除被日报或标准引用的配置项时由后端阻止破坏性删除。

### 12.2 工作当量标准

- 按业务分类、工作类型、产品系统和工作细项维护折算系数。
- 产品和工作细项可设置为“不限”，形成通用匹配规则。
- 支持新增、编辑、启用、禁用和删除标准。
- 已被日报引用的标准不能被直接破坏性删除。
- 标准按版本保存；日报保留实际使用的标准版本和系数来源。

### 12.3 Excel 标准导入

- 上传 Excel 工作当量标准文件并执行导入预检。
- 识别新增、重复、冲突和无效行。
- 预检不写入数据库，用户确认后才导入。
- 导入结果保留来源文件和标准版本信息。

### 12.4 分析规则

- 配置工作重心评分中工作当量、投入时间和记录数量的权重。
- 配置能力长期未记录天数和目标占比偏差阈值。
- 为各能力维度设置目标投入占比。
- 配置结果用于展板排序和成长提醒，不改变原始日报。

### 12.5 完整备份与恢复

- 下载包含主要业务表和元数据的完整备份包。
- 上传备份文件后先做结构、版本和数据量预检。
- 用户确认后才执行恢复。
- 恢复过程按数据库事务执行，失败时不保留半完成状态。

### 12.6 年度归档

- 选择年份并预览该年度记录、项目、成果等归档数量。
- 确认后生成年度归档文件。
- 归档用于长期保留年度数据，不自动删除当前数据库内容。

## 13. 报告生成与文件输出

- 工作台账、周报、月报和年报可生成标签分组文本报告。
- 报告弹窗支持查看并复制生成内容。
- Office 导出由后端生成，支持 `.docx`、`.pdf` 和 `.xlsx`。
- 导出范围和筛选条件写入导出请求，避免只在界面上筛选、文件仍导出全部数据。
- 年度 Excel 可输出原始工作当量与本次汇报折算值，同时保留两者口径说明。

## 14. 数据与运行保障

### 14.1 数据持久化与兼容

- 使用本地 SQLite 保存日报、项目、成果、能力分配、目标、里程碑、复盘、配置和标准。
- 数据库启动时执行增量迁移，并兼容早期日报字段。
- 业务关联使用稳定 ID；名称和别名主要用于展示与搜索。
- 关键写入使用事务，避免关系表只写入一部分。

### 14.2 数据口径保护

- 原始工作当量只在日报层计算一次。
- 项目、成果和报告读取同一日报原值。
- 多能力统计按比例分摊，多成果统计按日报去重。
- 非项目事项不计入项目数量和项目排行。
- 自动摘要、环比和提醒均能回到源日报或成果，不以无依据推断替代事实。

### 14.3 启动与验证

- 后端提供健康检查接口。
- 后端端口占用时输出可识别的启动错误，便于定位已有进程。
- 提供自动化测试、类型检查和前后端生产构建命令。
- 提供独立运行验证脚本，使用临时数据库检查启动、基础接口和典型数据量下的响应时间，不污染正式数据。

## 15. 当前主要数据关系

| 主体 | 可关联对象 | 关系用途 |
|---|---|---|
| 日报 | 项目、多个能力、成果 | 记录事实和唯一原始投入来源 |
| 项目 | 日报、成果 | 汇总投入、轨迹、进展和个人贡献 |
| 成果 | 项目、多个日报、多个能力、里程碑 | 形成可汇报且可追溯的成果证据 |
| 成长目标 | 多个里程碑 | 组织职业方向、培养计划和年度目标 |
| 里程碑 | 日报、项目、成果、阶段清单 | 自动或手工追踪量化进度 |
| 周/月/年复盘 | 周期日报、周期成果、手工复盘 | 形成数据判断、总结和下一步计划 |

## 16. 明确不应误解为已实现的内容

- 当前不是云端多用户系统，没有账号、权限和多人协作流程。
- 当前不保存附件和外部链接。
- 当前不自动把工作价值、成果价值或成长水平折算为综合评分。
- 当前不自动对传统业务应用固定折扣。
- 当前的 Excel 标准导入已具备预检和确认流程，但具体公司模板仍以实际文件字段兼容结果为准。
- 系统生成的是基于已有数据的汇报材料，不替代用户对背景、影响和未来计划的最终判断。
