# Trace 指标口径与来源规范

日期：2026-07-15

状态：Stage 1 已核对核心统计链路。本规范用于约束展板、复盘、导出和后续新增指标，避免把同期数据、间接关系或重复关系误写成直接投入与产出结论。

## 1. 基本原则

- 工作记录、原始工时和原始工作当量属于事实数据，派生指标不得回写事实。
- 同一记录在项目、业务和公司工作当量汇总中只计算一次。
- 多能力分析使用记录保存的分配比例；旧记录没有分配明细时，才在能力之间平均分配。
- 成果投入只能来自成果实际关联的工作记录，不能使用周期总投入除以成果数量推算。
- 多个成果引用同一记录时，单个成果可分别展示，跨成果汇总必须按记录 ID 去重。
- 系统可以陈述事实和规则提醒，不自动判断工作价值、效率或投入是否合理。

## 2. 核心指标口径

| 指标 | 来源 | 公式与范围 | 去重规则 | 允许表述 |
|---|---|---|---|---|
| 记录数 | 工作记录 | 当前筛选范围内记录数量 | 按记录 ID | 本周期记录 N 项 |
| 原始工作当量 | 工作记录 `workload` | 当前范围记录当量求和 | 按记录 ID | 本周期原始工作当量为 N |
| 投入工时 | 工作记录 `timeHours` | 当前范围工时求和，空值按 0 | 按记录 ID | 本周期记录投入 N 小时 |
| 项目投入 | 工作记录项目关系 | 按项目分组后汇总原始当量和工时 | 每个项目内按记录 ID | 投入主要集中在某项目 |
| 项目集中度 | 项目投入 | 首位项目当量 / 周期总当量；无当量时使用记录数 | 按记录 ID | 集中度为 N%，请确认是否符合当前重点 |
| 能力投入 | 能力分配 | 记录工时或当量乘能力分配比例 | 每条记录的能力比例合计 100% | 某能力分配后投入为 N |
| 能力占比 | 分配后能力投入 | 优先使用分配后当量占比；无当量时使用分配后工时，再无则使用分配权重 | 不重复增加记录原值 | 某能力投入占比为 N% |
| 成果数量 | 成果 | 当前周期内状态为阶段成果或已完成的成果数 | 按成果 ID | 本周期形成 N 项成果证据 |
| 成果关联投入 | 成果与记录关系 | 当前周期内，被成果直接关联的记录工时和当量求和 | 多成果共享记录按记录 ID 去重 | N 条记录提供 N 当量的直接投入证据 |
| 目标与里程碑进度 | 目标、里程碑及证据 | 按里程碑配置的数量、投入、阶段或持续口径计算 | 遵循各证据对象 ID | 当前证据支持进度 N% |
| 年度汇报折算 | 年度原始工作当量 | 原始当量乘本次用户填写比例 | 不写回任何来源数据 | 本次汇报折算结果为 N |

## 3. 提醒边界

- 可以提示“当前周期已有投入，但尚未记录阶段性或已完成成果证据”。
- 可以提示“成果未关联当前周期日报，无法计算成果关联投入”。
- 可以提示“项目投入集中度达到阈值，请确认是否符合当前重点”。
- 不得根据工时或工作当量直接得出“价值高”“效率低”“投入不合理”等结论。
- “高投入低产出”只能在同一项目或目标内，基于明确关联、配置阈值和足够周期判断；否则只能描述为证据缺失。

## 4. Stage 1 审计结论

本阶段确认并修正两类问题：

1. 复盘页原先使用周期总工作当量除以成果数量，无法证明是每项成果的实际投入。现改为只统计成果直接关联且位于当前周期内的记录，并按记录 ID 去重。
2. 展板、成长占比和导出原先会把多能力记录的全部工时和当量分别计入每个能力。现改为使用已保存分配比例；旧记录没有分配明细时平均分配。

本阶段不修改数据库结构、不重算历史记录、不改变项目、业务和公司原始工作当量。修正只发生在派生统计和展示层。

## 5. 新指标准入

后续新增指标前必须同时说明：

1. 它帮助用户做什么判断。
2. 使用哪些来源对象和时间范围。
3. 公式、去重和缺失数据处理方式。
4. 是否存在相关性被误写为因果关系的风险。
5. 如何返回来源记录、项目、成果或目标。
6. 对应的单元测试和历史兼容测试。

无法说明以上内容的指标不进入正式展板或报告。
