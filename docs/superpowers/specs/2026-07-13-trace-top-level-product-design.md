# Trace 顶层产品与功能板块设计

日期：2026-07-13

状态：已完成顶层使命、产品边界和七个功能板块的逐项确认，等待用户对本文档进行最终审阅。

## 1. 文档目的

本文档重新定义 Trace 的顶层使命、产品边界、核心数据关系和功能板块，作为后续需求整理、数据模型设计和分阶段实施的统一依据。

本文档描述目标产品，不代表其中全部功能已经实现。当前实现情况、后续开发优先级和迁移方案应在本文档确认后，结合现有代码另行形成实施计划。

## 2. 产品定位

Trace 是一个面向个人职业发展的工作量化、成果管理与成长复盘系统。

系统的核心使命是：

> 将日常工作事实持续沉淀为可量化、有关联、可复盘、可汇报的长期职业证据。

它不只是日报工具，也不只是工作当量统计工具。它应同时服务于：

- 记录个人成长和职业轨迹。
- 记录日常工作，识别当前工作重点和投入方向。
- 管理正式成果、重要问题解决、阶段性进展和可复用资产。
- 支持更细致的每周、每月回顾和未来规划。
- 为年底个人总结、工作当量统计和年度汇报提供完整材料。
- 支持个人职业复盘和公司培养计划回顾。
- 说明一个项目投入了多少工时与工作当量，并形成了哪些成果。

## 3. 需要解决的核心问题

Trace 应持续帮助用户回答以下问题：

1. 我在某一天、某一周、某个月和某一年具体做了什么？
2. 我的工时和工作当量主要投入到了哪些项目、业务、产品和工作类型？
3. 这些投入形成了哪些成果、阶段进展和关键问题解决？
4. 我在项目中承担了什么角色，实际贡献是什么？
5. 我的能力、责任范围和工作重心发生了什么变化？
6. 当前工作是否与公司培养方向和个人职业目标一致？
7. 下一个阶段最应该关注、改进和推进什么？
8. 年底如何快速获得有事实依据的个人总结、工作当量统计和汇报材料？

## 4. 产品边界

### 4.1 产品范围

- 面向单个用户的个人工作与职业发展管理。
- 以结构化事实记录为基础，以项目为主要关联主线。
- 同时保留公司工作当量口径和个人成长分析口径。
- 支持非项目型工作、学习、培训、认证和其他成长活动。
- 支持日、周、月、年四个时间层级的记录和复盘。
- 支持记录、项目、成果、能力、目标和报告之间相互追溯。

### 4.2 明确不做

- 不做团队项目管理和任务分派系统。
- 不做考勤、打卡或工时审批系统。
- 不做绩效审批、领导评价或组织排名系统。
- 不做多人账号、复杂权限和协同编辑。
- 不做通用笔记或完整知识库。
- 不保存附件和外部链接。
- 不以单一综合分数评价个人表现或成长。
- 不以图表数量作为数据展板的完成标准。
- 不自动判断工作价值，也不把高投入直接等同于高价值。

## 5. 设计原则

### 5.1 一次记录，多处复用

日常工作事实只录入一次，后续自动参与项目统计、成果追溯、能力分析、周报、月报、年报和导出，避免重复填写同一事实。

### 5.2 事实与总结分离

系统负责组织记录、投入、成果和变化；个人判断、原因分析、经验反思和未来规划由用户填写。报告不应把自动拼接的文字伪装成用户的真实复盘。

### 5.3 投入与产出并重

工时和工作当量描述投入，成果、问题解决和阶段进展描述产出。二者需要建立关联，但不能互相替代。

### 5.4 公司口径与个人口径分离

工作记录只保存一次原始工时和原始工作当量。能力分析可以按能力分配投入，但不能重复增加公司工作当量；年度汇报折算也不能改写历史原始数据。

### 5.5 历史稳定

工作当量标准、分类名称或规则发生变化时，已经保存的记录保留当时的系数、结果和标准版本。系统不得静默重算历史数据。

### 5.6 轻量录入

日报是主要数据入口，应通过搜索、联动选项、常用项、复制旧记录、自动匹配和手动草稿降低记录成本。

### 5.7 全程可追溯

报告中的数字、成果和结论能够返回来源记录、项目或目标。若来源数据错误，应修改来源，而不是在报告中直接覆盖统计值。

### 5.8 展板服务于行动

数据展板围绕工作重点、周期变化、投入产出和下一步行动组织。每个指标或提醒都应能够解释原因并查看来源。

## 6. 核心业务链

Trace 的核心链路为：

职业目标 → 项目 → 工作记录与投入 → 问题解决 / 阶段进展 / 正式成果 → 能力成长 → 周报执行回顾 → 月报成长复盘 → 年报总结与汇报

这条链路不是强制要求每条记录关联所有对象：

- 普通工作可以只保存为工作记录。
- 非项目工作可以明确标记为非项目事项。
- 重要记录可以关联一个或多个能力维度。
- 多条记录可以共同形成一个成果。
- 一个项目可以产生多个阶段进展、问题解决和正式成果。
- 一个目标或里程碑可以由多个项目、成果和记录共同支撑。

## 7. 核心对象及关系

### 7.1 工作记录

最小事实单元，描述某一天的一项工作。它保存分类、项目、数量、系数、原始工作当量、工时、工作内容、问题、结论、下一步行动和能力分配等信息。

### 7.2 项目

组织一段时间内相关工作、投入、进展、成果和个人贡献的主要载体。项目不是简单文本标签，而是可持续追踪的独立对象。

### 7.3 成果

描述工作形成的可识别产出，分为正式交付成果、重要问题解决、阶段性进展和可复用资产。成果可以关联一个项目和多条工作记录。

### 7.4 能力维度

描述工作对个人能力方向的锻炼和贡献。一条工作记录可以关联多个能力维度，但其工时和工作当量只能计算一次。

### 7.5 目标与成长里程碑

用于跟踪职业目标、公司培养目标、年度目标和阶段成长任务。目标通过项目、成果、记录和阶段确认获得进度依据。

### 7.6 周报、月报和年报

报告是来源数据在特定周期内的组织和复盘结果，不是另一套工作事实。自动统计部分来源于前述对象，手工复盘部分独立保存为报告草稿或正式内容。

## 8. 功能板块

系统采用面向业务对象的七板块结构。

### 8.1 今日工作台

#### 目标

快速记录当天工作，并立即建立与项目、工作当量、成果和能力成长的关系。

#### 功能

- 支持日期切换和当日概览。
- 录入标题、日期和项目；允许明确标记非项目事项。
- 选择业务分类、工作类型、产品、子任务和能力维度。
- 录入数量、计量单位、折算系数、原始工作当量和投入工时。
- 记录完成内容、遇到的问题、结论和下一步行动。
- 选项较多时支持关键词搜索。
- 工作类型、产品和子任务支持联动筛选。
- 支持常用选项和复制历史记录。
- 支持手动保存、恢复和清空草稿，不采用自动草稿保存。
- 保存后可以将一条记录提炼为成果。
- 显示折算系数的匹配来源和标准版本。
- 支持当日记录的查看、编辑和删除。

#### 工作当量匹配

- 系统根据工作类型、产品和子任务匹配标准。
- 优先使用条件最具体的规则。
- 只有标准库中明确存在通用规则时，才允许逐级匹配通用项。
- 无匹配结果时不伪造默认值，由用户手动填写。
- 修改数量或系数后重新计算原始工作当量。

#### 多能力分配

- 一条记录可以选择多个能力维度。
- 默认在所选能力之间平均分配工时和工作当量。
- 用户可以手动设置能力分配比例，比例合计必须为 100%。
- 能力分配只用于成长分析，不改变记录的原始工时和原始工作当量。

### 8.2 工作台账

#### 目标

作为全部工作事实的统一档案库，支持检索、核对、修正和向其他对象沉淀。

#### 功能

- 默认显示当前自然周记录。
- 支持周、月、年和自定义时间范围。
- 支持按关键词、项目、业务、工作类型、产品、子任务、能力、系数来源和是否产生成果筛选。
- 显示筛选范围内的记录数、工时、原始工作当量、项目数和成果数。
- 默认使用紧凑列表，展开后查看完整内容、能力分配、系数来源、标准版本和修改时间。
- 标记缺少项目、能力、工时、系数或工作内容的数据；明确的非项目事项不视为缺失项目。
- 支持编辑、删除和复制记录。
- 修改数量或系数时重新计算当前记录，并保留变更痕迹。
- 支持从一条或多条记录创建成果。
- 支持打开关联项目和成果。
- 支持把当前筛选结果交给复盘与汇报板块。

#### 边界

工作台账不承担报告撰写，也不对传统业务自动折扣。

### 8.3 项目管理

#### 目标

围绕项目聚合工作投入、阶段轨迹、成果价值和个人贡献。

#### 项目信息

- 项目名称和简称。
- 状态：计划中、进行中、暂停、已完成、已归档。
- 开始日期和结束日期。
- 业务方向和涉及产品。
- 个人角色。
- 项目目标和说明。

#### 功能

- 在日报中搜索、选择或快速创建项目。
- 支持项目改名、别名和重复项目合并。
- 按时间范围统计记录数、活跃天数、工时和原始工作当量。
- 展示业务、工作类型、产品和能力投入构成。
- 根据近期记录显示当前项目重点。
- 按时间线展示项目工作轨迹。
- 聚合阶段进展、重要问题解决、正式成果、可复用资产和成长里程碑。
- 已完成和已归档项目默认不出现在常用选择中，但可以搜索、查看和重新启用。
- 项目结束时支持填写角色、关键贡献、成果和经验，作为月报与年报材料。

#### 边界

项目管理不做团队成员分工、任务指派、审批和附件管理。项目只显示原始工作当量，不自动应用业务折扣。

### 8.4 成果管理

#### 目标

把散落在工作记录中的价值产出沉淀为可检索、可追溯、可复用的职业证据。

#### 成果类型

- 正式交付成果。
- 重要问题解决。
- 阶段性进展。
- 可复用资产。

原有知识资产概念并入可复用资产，不再单独扩展成通用知识库。

#### 状态

计划中 → 推进中 → 阶段成果 → 已完成

成果状态由用户维护，不设置审批流程。

#### 成果信息

- 标题、类型和状态。
- 可选关联项目。
- 开始、更新和完成日期。
- 背景与目标。
- 已完成内容或解决的问题。
- 价值与影响。
- 个人角色和具体贡献。
- 关联能力、目标和里程碑。
- 关联的一条或多条工作记录。
- 用于报告的简明表述。

#### 功能

- 从今日工作台、工作台账或项目中创建成果，也可直接创建非项目成果。
- 自动汇总关联记录的工时和原始工作当量。
- 按年份、项目、类型、状态、能力和关键词筛选。
- 详情中显示状态变化、来源记录、投入、能力和个人贡献。
- 可被周报、月报和年报引用。

#### 边界

成果价值不能只根据投入量判断，不提供主观分数、附件、外部链接和审批功能。

### 8.5 成长与目标

#### 目标

把公司培养要求、个人职业目标和能力成长连接到真实工作证据，同时避免把个人成长简化成一个总分。

#### 目标范围

- 职业发展目标。
- 公司培养目标。
- 个人年度目标。
- 学习、培训、认证和非项目成长事项。
- 可配置能力维度。

#### 里程碑量化方式

1. 数量型：按成果数、项目数或问题解决数计算；只有阶段成果和已完成成果计入。
2. 投入型：按关联能力的工时或工作当量计算；建议同时要求至少一个成果，避免只有投入没有产出。
3. 阶段型：以可勾选的阶段清单表示，例如认证准备、报名、考试和通过。
4. 持续型：按每月或每季度是否持续完成目标动作计算。

系统能够从记录、成果和项目计算的进度应自动计算；阶段型进度由用户确认。允许人工修正，但必须填写修正原因。

#### 功能

- 按月、季度和年度查看能力投入与变化。
- 查看每项能力关联的项目、成果和工作记录。
- 查看目标及里程碑的计划值、实际值、完成状态和证据。
- 记录学习、培训、认证和非项目成长。
- 提醒能力长期无投入、目标偏离、里程碑临期或逾期、投入较多但尚无成果、目标长期未更新等情况。
- 提醒必须说明触发原因，并能够跳转到相关记录、项目或成果。
- 相关内容进入周报、月报和年报。

#### 边界

目标与实际仅用于个人复盘，不等同于绩效评价。系统不生成个人成长总分。

### 8.6 复盘与汇报

#### 目标

把工作、项目、成果和成长数据转换为可回顾、可总结、可导出的汇报材料。

#### 共同规则

- 自动聚合选定周期内的来源数据，不重复保存工作事实。
- 区分数据视图和汇报视图。
- 支持与上一个周期或目标进行对比。
- 自动内容可以追溯到来源。
- 手工复盘内容支持保存、恢复和清空草稿。
- 若来源数据错误，从报告跳转到工作台账修改。

#### 周报

定位为短周期执行回顾，包含：

- 本周工作重点。
- 主要项目及投入。
- 工时和原始工作当量。
- 阶段进展和正式成果。
- 重要问题及解决情况。
- 未完成事项与原因。
- 下周重点和行动计划。

#### 月报

定位为阶段复盘和向上级汇报的主要材料，包含：

- 本月工作重点及变化。
- 各项目投入、进展和个人贡献。
- 正式成果、阶段进展和重要问题解决。
- 工时和原始工作当量构成。
- 能力投入和成长里程碑。
- 本月成绩与不足。
- 问题原因与改进措施。
- 新掌握的技能、工具和方法。
- 下月工作重点、成长计划和资源需求。
- 与上月和年度目标的对比。

#### 年报

同时服务于个人总结、工作当量统计、职业复盘和年度汇报，包含：

- 全年工作重点和轨迹。
- 各项目投入、进展、成果和个人贡献。
- 按业务、工作类型、产品、子任务和项目汇总工作当量。
- 重要成果、关键问题解决和代表性阶段进展。
- 能力成长、里程碑和职业目标完成情况。
- 各月份投入和重点变化。
- 高投入低产出、长期停滞等复盘线索。
- 下一年度重点和成长规划。

#### 年度汇报折算

- 系统各处默认保存和展示原始工作当量。
- 年报生成时可以填写汇报折算系数，默认 100%。
- 只有用户主动填写其他比例时，才生成折算后的汇报结果。
- 折算结果只用于本次报告预览和导出。
- 折算不得写回工作记录、项目统计或历史汇总。
- 传统业务不自动采用 85% 折扣。

#### 导出

- Word 和 PDF：个人总结、周报、月报和年度汇报。
- Excel：工作记录明细、项目投入、成果清单和年度工作当量表。
- 导出前可以选择时间、项目、业务分类和内容范围。
- 年度 Excel 后续应支持公司的工作当量统计表结构。

#### 展板

展板重点呈现核心汇总、周期变化、投入集中度、投入产出关系、能力进展和需关注事项。避免重复图表，所有统计均可下钻到记录。

### 8.7 配置与数据

#### 目标

维护分类、工作当量标准、个人偏好和数据安全，保证系统长期使用时口径一致、历史稳定。

#### 工作当量标准库

标准结构为：

业务分类 → 工作类型 → 产品 → 子任务 → 计量单位 → 折算系数

标准库支持新增、编辑、排序、启用和停用。匹配规则与今日工作台保持一致，并向用户显示自动匹配、通用规则或手动填写等来源。

#### 标准版本

- 标准按年度或版本管理。
- 启用新版本只影响新记录。
- 每条记录保存实际系数、系数来源和标准版本。
- 标准变化不自动重算历史记录。
- 如历史记录填写错误，用户可以在原记录中主动重新匹配，并保留修改痕迹。
- 标准库不设置传统业务统一折扣。

#### Excel 标准导入

系统后续支持把“技术部工作当量统计表-杨川.xlsx”中的“工作当量标准”工作表导入为当年的初始标准库：

- 导入前预览识别结果。
- 显示新增、重复、冲突和无法识别的数据。
- 用户确认后才正式写入。
- 冲突项支持保留系统值、采用 Excel 值或逐条处理。
- 导入完成后形成独立标准版本并记录来源说明。

Excel 字段映射和兼容策略留到该功能的实施规格中确定，不能直接假设工作表结构永远不变。

#### 基础选项

支持维护业务分类、工作类型、产品、子任务、能力维度、成果类型、项目状态和计量单位。每个选项支持新增、改名、排序、停用和搜索别名。

已经被历史数据引用的选项不能直接删除，只能停用或合并。合并必须保留旧值到新值的映射，以保证历史查询和显示正常。

#### 个人偏好

- 每周起始日。
- 默认统计周期和默认年度。
- 常用项目和常用选项。
- 周报、月报和年报的显示栏目。
- 数字精度和工作当量小数位。

年度汇报折算系数不作为长期自动规则保存，每次生成年度报告时由用户确认。

#### 备份与迁移

- 导出完整数据备份，用于恢复或迁移。
- 恢复前校验备份并预览影响范围。
- 支持以 Excel 导出工作记录、项目、成果、目标和工作当量。
- 支持按年度归档，但归档不删除数据。
- 删除重要数据前二次确认。
- 系统内部不保存附件或外部链接。

#### 数据质量检查

系统可以提示未匹配标准、手动系数占比偏高、缺少必要关联、重复分类、疑似重复项目、长期未备份以及标准库冲突或失效等问题。检查结果只提醒，不自动修改数据。

## 9. 报告层级与职责

### 9.1 日记录事实

记录当天发生的工作、投入、问题和行动，不承担周期总结。

### 9.2 周报回顾执行

关注短周期的工作、项目、工作当量、问题和下周行动，内容应简洁、便于快速汇报。

### 9.3 月报复盘阶段

关注工作重点变化、阶段成果、个人贡献、能力成长、问题原因和下月规划，是公司培养沟通和向上汇报的主要载体。

### 9.4 年报总结全年

关注全年轨迹、代表性项目与成果、工作当量、职业成长、经验教训和下一年度规划，为个人总结和正式汇报提供依据。

## 10. 关键计算口径

### 10.1 原始工作当量

原始工作当量 = 数量 × 实际折算系数

实际折算系数可以来源于标准自动匹配、明确的通用规则或用户手动输入。系统必须保存实际使用值和来源，而不是在查询时根据最新标准重新推导。

### 10.2 能力投入

当一条记录关联多个能力时：

- 默认平均分配该记录的工时和原始工作当量。
- 可以手动配置各能力比例，合计必须为 100%。
- 能力汇总使用分配后的数值。
- 项目、业务和公司工作当量汇总仍使用记录原值，不因能力数量重复计算。

### 10.3 成果投入

成果汇总其关联记录的工时和原始工作当量。同一条记录可以支撑多个成果时，成果详情可以分别展示该记录，但跨成果统计不得简单相加后当作无重复的总投入；汇总页面应按记录去重。

### 10.4 里程碑进度

- 数量型、投入型和持续型优先从来源数据自动计算。
- 阶段型由用户确认阶段完成情况。
- 人工修正保留修正值、原因和时间。
- 进度用于复盘，不转换成统一个人评分。

### 10.5 年度汇报折算

汇报折算值 = 年度原始工作当量 × 用户本次填写的汇报折算系数

该值是报告层的临时派生结果，不属于工作记录事实。

## 11. 数据流

1. 用户在今日工作台录入事实，系统匹配或接收折算系数并保存原始工作当量。
2. 工作记录进入工作台账，并关联项目和一个或多个能力维度。
3. 用户从记录、项目或成果管理中沉淀阶段进展、问题解决、正式成果或可复用资产。
4. 项目聚合相关记录、投入、时间线、成果和成长证据。
5. 成长与目标根据能力分配、成果和项目关系计算目标与里程碑进展。
6. 周报、月报和年报按周期读取来源数据，生成统计与证据清单。
7. 用户补充复盘判断和未来计划，再预览或导出报告。
8. 所有汇总可以返回来源对象，来源修正后重新生成统计。

## 12. 异常与边界处理

- 当量标准无匹配时，明确提示手动输入，不能静默使用 0、1 或最近一次系数。
- 多个标准同时匹配且优先级相同时，停止自动填充并提示标准冲突。
- 能力分配比例不等于 100% 时不能保存手动分配，可恢复为平均分配。
- 删除被引用的项目、分类、能力或标准时，改为停用、归档或合并，不直接破坏关联。
- 删除工作记录或成果前，提示会影响的项目、目标和报告统计。
- 报告没有来源数据时显示空状态，不能生成看似完整但无事实依据的内容。
- 草稿保存失败不应覆盖已保存的正式报告内容，并应明确反馈。
- 导入前必须校验结构和数据类型；导入失败不得留下半完成版本。
- 备份恢复前进行完整性检查，并在覆盖现有数据前二次确认。
- 金额以外的数值精度遵循个人设置，但数据库保留足够计算精度，显示舍入不能反向改写原值。

## 13. 数据展板原则

每个周期展板应优先回答四类问题：

1. 重点：时间和工作当量主要投入在哪里？
2. 变化：与上一个周期相比，投入、项目和成果发生了什么变化？
3. 产出：投入形成了哪些结果，哪些事项仍处于高投入低产出状态？
4. 行动：哪些项目、能力或目标需要下一步关注？

展板不要求固定图表数量。新增图表前必须说明它支持的判断和后续行动；不能支持判断的装饰性图表不进入正式范围。

## 14. 验收原则

### 14.1 业务闭环

- 一条工作记录可以完整参与项目、成果、能力和报告链路。
- 一个项目可以清楚看到投入、轨迹、成果和个人贡献。
- 一项成果可以追溯到来源记录，并进入周期报告。
- 一项目标或里程碑可以看到计算口径和支撑证据。
- 年报能够产出个人总结材料和年度工作当量明细。

### 14.2 数据正确性

- 标准匹配、手动系数、工作当量、能力分配和报告折算均有独立测试。
- 同一记录在项目、成果、能力和报告中的汇总口径一致。
- 多能力和多成果关系不会导致原始工时或工作当量重复计算。
- 标准版本变化不会改写历史结果。
- 导入、备份和恢复具备失败回滚或原子性保证。

### 14.3 可用性

- 常用日报在较少操作内完成录入。
- 长选项可以搜索，联动选项不会显示大量无关内容。
- 周报、月报和年报既能看统计，也能编辑个人复盘内容。
- 数据异常和缺失项能够被发现并定位到来源。
- 紧凑列表和按需展开保证多年数据仍可检索。

### 14.4 回归验证

每个实施阶段至少覆盖：

- 核心计算和筛选的单元测试。
- 前后端接口和数据库迁移测试。
- 关键用户流程测试。
- 历史数据兼容测试。
- 完整测试与生产构建。
- 报告导出和关键页面的人工验收。

## 15. 实施拆分原则

本设计覆盖多个相互关联的子系统，不应一次性整体重写。后续实施计划应按依赖关系拆分，建议顺序为：

1. 统一核心对象和历史数据兼容策略。
2. 完善项目对象、标准版本和日报数据入口。
3. 建立成果对象及记录、项目、成果关系。
4. 完善工作台账和数据质量检查。
5. 完善成长目标、能力分配和里程碑量化。
6. 重构周报、月报、年报及展板口径。
7. 完成 Excel 标准导入、年度工作当量导出、备份与恢复。

每个阶段都应形成独立的实施规格、迁移方案和验收标准，避免把顶层设计直接转换成一次大规模开发任务。

## 16. 已确认的关键决策

- 产品以个人职业发展为目标，不扩展为团队管理系统。
- 采用今日工作台、工作台账、项目管理、成果管理、成长与目标、复盘与汇报、配置与数据七个板块。
- 增加周报层级，形成日、周、月、年四层记录与复盘。
- 成果同时包含正式交付、重要问题解决、阶段性进展和可复用资产。
- 不保存附件和外部链接。
- 一条工作记录可以关联多个能力，默认平均分配，可手动设置比例。
- 公司原始工作当量只计算一次，不因多能力重复。
- 成长里程碑采用数量型、投入型、阶段型和持续型四种量化方式。
- 不采用统一个人成长总分。
- 传统业务不自动打折；需要时在年度报告中由用户主动输入汇报折算系数。
- 工作当量标准支持按工作类型、产品和子任务自动匹配，无匹配时手动填写。
- 后续将现有 Excel 的“工作当量标准”工作表导入为初始标准库，但字段映射在实施阶段单独确认。

## 17. 成功标准

当用户持续使用一年后，Trace 应能在较少额外整理工作的情况下：

- 准确说明全年做过的主要工作和工作重点变化。
- 按项目、业务、产品和工作类型统计原始工作当量与工时。
- 展示每个重点项目的投入、进展、成果和个人贡献。
- 列出全年正式成果、关键问题解决、阶段进展和可复用资产。
- 结合真实证据复盘能力成长、培养目标和职业目标。
- 快速形成周报、月报、个人年度总结、年度工作当量表和向上汇报材料。
- 从数据中识别下一阶段应保持、调整或优先推进的方向。
