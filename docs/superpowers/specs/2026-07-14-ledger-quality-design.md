# Trace 工作台账与数据质量设计

日期：2026-07-14

状态：已确认，进入实施

## 1. 目标

阶段 3 把现有“全部记录”升级为可长期维护的工作事实档案。页面需要帮助用户快速定位多年记录、核对投入口径、发现数据缺口，并把筛选结果继续沉淀为成果或报告材料。

本阶段不改变工作当量计算，不自动修复数据，不新增导航板块，也不建设通用数据治理系统。

## 2. 实施范围

### 2.1 组合筛选

- 时间范围支持当前周、指定周、指定月、指定年和自定义起止日期。
- 维度支持关键词、项目关系、业务分类、工作类型、产品、子任务、能力、系数来源、成果关联状态和标签。
- 所有条件采用交集口径；清除筛选恢复当前自然周。
- 筛选范围统计记录数、工时、原始工作当量、项目数和成果数。

### 2.2 紧凑台账

- 默认以紧凑行展示日期、标题、项目、分类、工时、工作当量和质量状态。
- 单行展开后显示完整工作内容、能力分配、系数来源、标准版本、标签和更新时间。
- 支持单条及多选；选择结果可批量提炼为一个成果。
- 生成报告时优先使用已选择记录；没有选择时使用当前筛选结果。

### 2.3 数据质量

每条记录按确定性规则派生质量问题：

- 项目关系为 `unassigned` 时提示缺少项目归属；明确的非项目事项不提示。
- 能力分配为空时提示缺少能力。
- 工时为空时提示缺少工时。
- 系数来源为 `none` 时提示缺少系数。
- 工作内容为空时提示缺少工作内容。

范围级诊断包括：

- 手动系数记录数量和占比。
- 对业务、工作类型、产品和子任务按“去空格、标点及大小写”归一化后，识别名称不同但归一值相同的疑似重复分类。
- 对未合并项目使用同一归一化规则识别疑似重复项目。

质量提醒只用于筛选、定位和解释，不自动修改源数据。点击提醒后进入对应问题筛选；记录修改仍使用现有编辑流程。

### 2.4 删除影响

删除记录前调用影响预览接口，显示其关联项目和成果。确认文案说明删除后将影响这些对象的投入和报告统计；未关联时仍执行普通二次确认。

“清空全部记录”保留，但必须显示记录数量及会影响所有项目、成果和报告统计的明确警告。

## 3. 数据与接口

- 筛选、范围统计和质量诊断由前端纯函数从已加载记录、项目和成果计算，不新增持久化质量表。
- 新增 `GET /api/records/:id/impact`，返回记录基本信息、关联项目快照及关联成果简表。
- 项目和成果使用现有 API 加载；失败时页面仍展示台账，相关筛选显示错误提示而不是阻塞记录浏览。
- 跨成果统计按记录去重；成果数按成果 ID 去重。

## 4. 页面结构

1. 页头：备份、生成报告、清空数据。
2. 范围指标：记录、工时、原始工作当量、项目、成果。
3. 筛选带：时间模式和组合筛选，支持折叠次要条件。
4. 质量带：缺失项、手动系数和疑似重复提醒，点击即可筛选。
5. 批量操作带：已选数量、提炼成果、生成报告、清除选择。
6. 紧凑记录列表：稳定列宽，按需展开详情。

视觉继续使用 Trace 已确认的浅色、紧凑、工作型界面；质量状态使用语义色和图标，不增加装饰性图表或卡片嵌套。

## 5. 异常与边界

- 无记录时展示可行动空状态。
- 起始日期晚于结束日期时不执行筛选并显示字段错误。
- 已选择记录在筛选后不可见时从选择集合中移除，避免对隐藏记录误操作。
- 多选记录属于不同项目时，创建成果不预填项目，只预填记录 IDs。
- 旧记录的 `legacy` 系数来源不视为缺失，也不计入手动系数占比。
- 同一记录同时存在多个质量问题时只计一次“有问题记录”，但各问题计数分别累加。

## 6. 验收标准

- 当前周、周、月、年、自定义范围以及所有组合维度具有独立过滤测试。
- 质量规则、重复识别、手动系数占比和统计去重具有纯函数测试。
- 删除影响接口覆盖无关联、项目关联、多个成果关联和不存在记录。
- 可以从筛选结果多选记录创建成果或生成报告。
- 桌面和移动端均可完成筛选、展开、编辑、删除和批量操作。
- 完整测试、类型检查和生产构建通过。
