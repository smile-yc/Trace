# Trace 阶段 5：复盘、报告与行动型展板实施计划

## 目标

建立“日报事实、周报执行、月报复盘、年报总结”的报告层。自动统计只从源数据计算，手工内容独立保存，所有判断能够回看来源记录。

## 交付范围

- 周、月、年报告统一支持数据视图与汇报视图。
- 手工复盘字段：成绩、不足、原因、改进、成长、下一周期规划；支持保存草稿和确认定稿。
- 自动判断：环比变化、项目投入集中度、投入产出、下一步提醒。
- 统计结论提供源记录或成果下钻，不允许直接修改自动统计。
- 年报支持临时工作当量折算比例，默认 100%，仅影响当次预览和导出；原始值始终保留。
- 月报与年报原始明细继续沿用紧凑归档，完整周期分析不被截断。

## 数据模型

新增 `report_reviews`：以 `reportType + periodKey` 唯一标识一份报告，保存六类手工复盘文本、草稿/定稿状态和更新时间。自动统计、比较结果和临时折算比例不写入数据库。

## 实施顺序

1. 以数据库测试建立报告复盘的读取、保存、定稿和重新编辑能力。
2. 建立纯函数报告分析，覆盖周期变化、集中度、投入产出和提醒。
3. 实现共享复盘编辑器和判断面板，并接入周报、月报、年报。
4. 实现年度临时折算预览及 Office 导出说明，保持明细原值。
5. 运行测试、类型检查、构建和 API 冒烟检查；仅合并本地 `main`。

## 完成标准

- 自动统计与手工复盘在界面和数据结构上明确分开。
- 三类报告均可保存草稿、定稿并重新读取。
- 周期变化、集中度和投入产出均能展开来源证据。
- 年度折算不写回记录，导出同时标明原始值和折算值。
- 传统业务不自动打折。
