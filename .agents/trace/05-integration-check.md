# Agent 5：集成检查

## 角色定位

集成负责人、冲突审查者和最终质量门禁。

## 当前波次

贯穿全部波次。第一次介入发生在 Agent 1、Agent 2 交付后；第二次介入发生在 Agent 3、Agent 4 交付后。

## 具体目标

在不重写业务实现的前提下完成共享入口接线、冲突检查、跨域数据口径核对、自动化回归、桌面与移动端检查以及打印导出验收。

## 允许修改

- `frontend/src/App.tsx`
- `frontend/src/main.tsx`
- `frontend/src/navigation/**`
- 必要的共享类型入口和样式导入入口
- `backend/src/index.ts`
- 必要的迁移和后端路由注册入口
- 根级测试脚本和确有必要的集成测试
- 由合并冲突直接导致的小范围修正

## 禁止修改

- 未复现问题就大规模重写 Agent 3 或 Agent 4 的业务实现。
- 删除断言、跳过测试或降低类型约束。
- 修改 `.gitignore` 或任何无关用户文件。
- 擅自改变产品边界、统计口径、颜色或图表定义。
- 自行增加未经设计确认的新功能。

## 集成职责

1. 核对每个 agent 的提交和交接报告。
2. 检查实际修改是否符合文件所有权。
3. 注册页面、导航、后端路由、迁移和样式入口。
4. 验证 `EvidencePeriodSnapshot` 的前后端一致性。
5. 运行完整测试、类型检查和构建。
6. 验证多能力、多成果、人工系数、标准版本和年度临时折算。
7. 真实浏览器检查桌面、移动端和 A4 打印。
8. 输出问题清单；阻断问题必须退回责任 agent 修复并重新审查。

## 验收标准

- `pnpm run test`、`pnpm run typecheck`、`pnpm run build` 全部通过。
- 旧数据可读取，标准变化不重算历史。
- 七个模块可以访问并相互跳转。
- 统计、图表、报告和导出使用一致口径且可追溯来源。
- 桌面和移动端不存在重叠、溢出、空白图表或不可操作控件。
- 最终报告包含自动测试结果、检查视口、打印结果、剩余风险和合并建议。
