# Trace UI 设计规范

日期：2026-07-13

状态：已完成逐项讨论与确认，等待用户对书面规范进行最终审阅。

## 1. 文档目的

本文档定义 Trace 目标产品的界面结构、视觉语言、通用组件、核心页面、数据展板、响应式、可访问性和打印规则，作为后续 UI 实施与多 agent 协作的统一依据。

本文档描述目标体验，不表示当前代码已经实现全部内容。功能范围、数据对象和计算口径以《Trace 顶层产品与功能板块设计》为上位依据；若两份文档发生冲突，先遵循顶层产品设计，再补充修订本规范。

## 2. UI 目标

Trace 的 UI 需要同时服务两种使用状态：

1. 高频执行：快速记录工作、检查今日重点、维护项目进展和成果。
2. 深度阅读：完成周报、月报、年度总结、职业复盘和正式汇报。

因此，整体采用“冷静专业工作台 + 汇报级阅读体验”的方向：

- 执行页面紧凑、直接、便于连续操作。
- 报告页面留白更充足，内容层级接近正式文档。
- 不通过大量装饰、卡片或图表制造复杂感。
- 任何汇总数字、图表和提醒都应能够返回来源记录。
- 数据展板优先帮助判断重点、变化、产出和下一步行动。

## 3. 视觉原则

### 3.1 视觉性格

界面应呈现可靠、克制、清晰、长期可用的个人职业工作台，而不是营销网站、团队管理后台或游戏化成长产品。

禁止以下视觉倾向：

- 大面积渐变、发光和装饰性色块。
- 过多悬浮阴影和大圆角卡片。
- 用不同颜色包装每一个普通信息块。
- 用综合分数、排行榜、仪表盘或勋章评价个人表现。
- 用英文眉题、口号或说明性文案占据主要空间。
- 为了填满页面而增加无行动价值的图表。

### 3.2 色彩体系

品牌色、语义色和图表色必须分离，避免同一种颜色同时承担品牌、状态和数据系列三种含义。

基础色：

| 用途 | 色值 |
| --- | --- |
| 页面背景 | `#F5F6F7` |
| 侧边栏背景 | `#FAFBFB` |
| 主表面 | `#FFFFFF` |
| 次级表面 | `#F0F3F4` |
| 边框 | `#D9E0E3` |
| 标题 | `#182126` |
| 正文 | `#3D484F` |
| 辅助文字 | `#68747C` |
| 弱化文字 | `#89939A` |

品牌色：

| 用途 | 色值 |
| --- | --- |
| 主品牌色 | `#176B68` |
| 悬停或按下 | `#125A57` |
| 选中背景 | `#E7F2F0` |

语义色：

| 用途 | 色值 |
| --- | --- |
| 信息 | `#356D9F` |
| 成功 | `#3E7C52` |
| 警告 | `#A96B19` |
| 危险 | `#B54747` |

语义状态必须同时使用图标或文字，不得只依赖颜色。图表使用独立的数据色板，并保持同一项目、能力或成果类型跨页面颜色一致。

初期不提供深色模式。现有伪深色样式应在实施时移除，避免形成无法完整维护的第二套主题。

### 3.3 字体与数字

- 中文界面使用系统无衬线字体栈，保证 Windows 环境稳定显示。
- 常规字重限定为 `400`、`500`、`600`、`700`。
- 页面标题保持克制，紧凑面板内不使用英雄级字号。
- 字间距保持 `0`，不使用负字间距。
- 工时、工作当量、百分比和日期等数字采用等宽数字特性。
- 报告预览可以提高行高和段落间距，但不得改变基础字体体系。

### 3.4 形状、边框与阴影

- 普通控件和卡片圆角为 `6px` 至 `8px`。
- 页面区块默认不做悬浮卡片，使用分区、边框和留白建立层级。
- 阴影仅用于弹窗、菜单和确有层级关系的浮层。
- 不使用渐变背景。
- 卡片只用于汇总信息、重复项目、工具面板和弹窗，不允许卡片嵌套卡片。

## 4. 全局布局与导航

### 4.1 桌面布局

- 固定侧边栏宽度为 `216px`。
- 主工作区自适应，占用剩余空间。
- 页面左右留白根据宽度保持 `24px` 至 `32px`。
- 页面不再包裹在一个整体悬浮卡片中。
- 数据列表可以使用宽布局；报告预览内容宽度控制在约 `960px`。

### 4.2 品牌区

品牌图标采用“轨迹、路径、节点”语义，不使用数据库图标。品牌区只显示 Trace 名称和必要的当前年度或工作区信息，不加入宣传口号。

### 4.3 导航结构

侧边栏按业务分组：

| 分组 | 页面 |
| --- | --- |
| 记录 | 今日工作台、工作台账 |
| 工作 | 项目管理、成果管理 |
| 成长 | 成长与目标 |
| 复盘 | 复盘与汇报 |
| 系统 | 配置与数据 |

“配置与数据”固定在侧边栏底部。导航使用图标和中文名称，当前页面采用品牌色文字、浅色选中背景和明确的选中标记。

页面切换后保留各页面最近一次筛选条件和滚动位置，避免在台账、项目和报告之间往返时反复操作。

### 4.4 页面标题区

所有页面使用统一的紧凑标题区：

- 左侧显示页面标题、必要状态和一句上下文信息。
- 右侧显示周期选择、次要操作和最多一个主要操作。
- 页面标题不重复解释功能，不添加英文眉题。
- 筛选器、标签页和页面标题之间保持稳定间距。

## 5. 通用组件与交互

### 5.1 按钮

按钮分为主要、次要、文字和危险四种。

- 每个页面标题区最多一个主要按钮。
- 明确命令使用图标加文字，例如“保存记录”“新建成果”。
- 撤销、关闭、更多、删除等熟悉操作可以使用纯图标，必须提供悬停说明。
- 桌面常规按钮高度为 `36px`，移动端触控高度不低于 `44px`。
- 危险操作不得因视觉弱化而隐藏影响范围。

### 5.2 表单

- 标签位于输入框上方。
- 工作类型、产品、子任务采用可搜索联动选择器。
- 项目、能力和成果等长选项均支持关键词搜索。
- 数值输入显示单位，并明确小数精度。
- 校验错误显示在对应字段附近，不只使用顶部提示。
- 多能力默认平均分配，切换到手动模式后显示各能力比例，合计必须为 `100%`。
- 复杂表单通过标题、说明和分隔线分区，不使用多层卡片。

长表单底部使用稳定操作区。工作记录提供“清空草稿”“保存草稿”“复制上一条”“保存记录”；不采用自动保存。正式保存后清除对应草稿。

### 5.3 筛选栏

默认展示关键词、时间范围和两个最常用筛选项，其余放入“更多筛选”。

- 筛选栏在长列表滚动时保持可访问。
- 已启用筛选以可移除条件标签显示。
- 提供“清除全部”。
- 时间范围统一支持本周、本月、本季度、本年和自定义。
- 图表点击形成的筛选必须明确显示，并可一键恢复。

### 5.4 状态与标签

- 状态使用图标、文字和浅色语义背景。
- 项目、业务、产品、能力等分类使用中性标签，不把普通分类做成彩色状态。
- 同一状态跨页面保持相同名称、图标和颜色。

### 5.5 表格与列表

- 桌面表格行高为 `44px` 至 `48px`。
- 表头可吸顶，数字右对齐，文字保持可扫描。
- 默认列只显示决策所需信息，完整信息进入详情。
- 行点击打开详情侧栏，编辑复杂对象时进入完整页面或完整表单。
- 移动端将宽表格转换为信息层级明确的记录列表。

### 5.6 反馈与异常

- 保存成功使用短时提示。
- 表单错误就地显示。
- 页面加载失败显示原因、影响和重试操作。
- 加载使用稳定骨架或占位，避免布局跳动。
- 删除、恢复、导入等高风险操作必须说明影响范围并要求确认。
- 空状态直接给出下一步操作，不展示装饰性插画或冗长教程。

### 5.7 详情承载方式

- 工作记录、项目摘要和成果摘要优先使用右侧详情栏。
- 长表单、项目完整详情、成果完整详情和报告编辑使用独立页面。
- 移动端详情统一进入全屏页面。

## 6. 今日工作台

### 6.1 页面结构

页面顶部显示日期和当日简要状态，随后是横向摘要带。桌面主体采用约 `65% : 35%` 的双栏结构：左侧为记录表单，右侧为当日记录；窄屏转为单栏。

摘要带仅显示当日记录数、工时、原始工作当量、涉及项目和需要补充的信息，不做装饰性指标卡矩阵。

### 6.2 记录表单

表单分为三段：

1. 工作内容：标题、日期、项目或非项目事项、完成内容。
2. 投入与分类：业务、工作类型、产品、子任务、数量、单位、系数、原始工作当量、工时和能力分配。
3. 复盘与关联：问题、结论、下一步行动以及是否提炼为成果。

系统自动匹配系数时显示匹配来源和标准版本；无匹配时明确进入人工系数状态；规则冲突时停止自动填充并提示处理。传统业务不在此处自动打折。

### 6.3 当日记录

当日记录采用紧凑列表，支持编辑、复制、提炼为成果和删除。详情在右侧区域或详情栏展示，不为每条记录创建大型卡片。

## 7. 工作台账

### 7.1 默认视图

默认显示当前自然周，支持周、月、年和自定义周期。标题区下方依次为筛选栏、汇总带和记录表格。

汇总带显示记录数、工时、原始工作当量、项目数和成果数，并允许进入来源明细。

### 7.2 表格字段

默认列为：选择、日期、标题、项目、工作类型、工时、原始工作当量、系数来源、成果状态和操作。

行点击打开右侧详情栏，展示完整事实、系数来源、标准版本、能力分配、成果关系和修改痕迹；编辑进入完整记录表单。

### 7.3 批量操作

允许从多条记录创建成果或形成报告材料。不得批量修改项目、系数或工作当量，以免破坏历史口径。

## 8. 项目管理

### 8.1 项目列表

项目首页采用紧凑列表，不采用默认看板。提供状态标签页、关键词搜索、业务方向和时间筛选。

项目列表重点展示状态、当前重点、近期活动、投入、成果数量和下一步行动。项目进度不使用任意填写的百分比，而由状态、关键节点和证据表达。

### 8.2 项目详情

项目详情使用完整页面。顶部显示项目名称、状态、角色、周期和主要操作，随后显示记录数、活跃天数、工时、原始工作当量和成果数。

详情包含四个标签页：

1. 总览：目标、当前重点、角色、下一步行动和核心汇总。
2. 工作轨迹：按时间线展示工作记录和关键节点。
3. 成果与进展：聚合正式成果、重要问题解决、阶段性进展和可复用资产。
4. 项目复盘：记录关键贡献、经验、问题和结束总结。

项目结束时引导填写角色、关键贡献、代表成果和经验。项目改名保留稳定标识和全部历史关联。

## 9. 成果管理

### 9.1 成果首页

成果首页提供类型、状态、项目、能力、年度和关键词筛选，默认使用紧凑列表；可选按状态分组，但不使用默认看板。

成果类型通过图标和文字区分，不使用大面积颜色：正式交付成果、重要问题解决、阶段性进展、可复用资产。

### 9.2 成果详情

成果详情包含：

- 背景与目标。
- 已完成内容或解决的问题。
- 价值与影响。
- 个人角色和具体贡献。
- 关联能力、目标和成长里程碑。
- 来源工作记录、状态历史和投入汇总。
- 可直接用于报告的简明表述。

成果可以从今日工作台、工作台账、项目详情创建，也可以直接创建非项目成果。

删除成果不删除来源工作记录。同一记录支撑多个成果时，各成果详情可以展示该记录，但跨成果汇总必须按记录去重。长期无更新的阶段性进展应提示用户复核状态。

## 10. 成长与目标

模块包含“目标”“能力成长”“成长里程碑”三个视图。

### 10.1 目标

- 支持年度、季度和月度目标。
- 记录目标描述、完成标准、优先级、周期、状态和下一步行动。
- 可关联项目、工作记录、成果和成长里程碑。
- 不允许缺乏依据的任意完成百分比。
- 进度以完成标准、关联证据和当前状态表达。
- 优先提醒即将到期、长期无进展、偏离目标和缺少下一步行动的事项。

### 10.2 能力成长

能力分类由用户维护，不强制固定模型。一条工作记录可以关联多个能力，默认平均分配，也可以手动分配比例。

每项能力展示分摊后的工时和工作当量、关联项目、成果、重要问题解决及成长里程碑。系统不生成能力总分、排名或雷达图，而是说明能力在哪些场景中被使用、形成了什么证据以及下一步如何提升。

### 10.3 成长里程碑

成长里程碑必须包含时间、能力、具体变化和支撑证据。典型里程碑包括首次独立完成、角色变化、关键成果、重要突破和持续行为达成。

里程碑采用年度时间轴展示，并支持按能力和项目筛选。其内容可以直接进入月报、年报和职业复盘。

## 11. 复盘与汇报

模块包含周报、月报、年度总结和职业复盘。

### 11.1 报告工作区

报告工作区区分“数据分析”和“报告内容”两个视图。数据分析用于识别重点和变化；报告内容用于组织可提交文字。

报告支持草稿和已完成状态，并保留历史版本。手工复盘内容采用手动保存，不使用自动保存。来源数据发生错误时，从报告跳转到工作台账修正，再重新生成统计。

### 11.2 周报

周报包含本周工作重点、项目及阶段进展、成果和重要问题解决、工时与工作当量、本周成长、风险与待协调事项以及下周计划。

默认图表为工作量趋势和工作重点构成。

### 11.3 月报

月报包含本月重点及变化、重点项目与个人贡献、成果与重要问题、工作量结构、能力投入、目标推进、本月复盘和下月规划。

默认图表为工作量趋势、工作重点构成、成果产出趋势和周期对比。

### 11.4 年度总结

年度总结包含全年轨迹、项目投入与贡献、全年工作当量、代表成果、重要问题解决、能力成长、成长里程碑、职业复盘和下一年度规划。

年度工作当量折算只在生成报告时使用本次由用户填写的临时系数，默认 `100%`，不写回工作记录或项目汇总。

默认图表为年度工作量趋势、项目月度热力图、工作量分类结构、成果产出趋势、能力证据图和成长里程碑时间轴。

## 12. 数据展板与图表

### 12.1 展板顺序

数据展板按以下顺序组织：

1. 重点：主要时间和工作当量投入在哪里。
2. 变化：与上一周期相比发生了什么变化。
3. 产出：形成了哪些成果，哪些事项投入较多但证据不足。
4. 行动：哪些项目、目标或能力需要下一步关注。

每个图表都必须能够说明其支持的判断，并允许查看来源。能够用数字或列表清楚表达的内容，不强行转换为图表。

### 12.2 图表目录

#### 工作量趋势图

柱状图或折线图，按日、周或月展示工时或原始工作当量。通过分段控件切换指标，不使用双坐标轴。

#### 工作重点构成图

使用 `100%` 堆叠条形图，展示项目、工作类型或产品的投入占比，用于识别重心变化。

#### 项目投入轨迹图

使用时间趋势图展示项目的周期投入，并在图上标记阶段性进展、重要问题解决和正式成果。

#### 项目月度热力图

横轴为月份，纵轴为项目，色深表示投入，用于年末识别持续项目、阶段性项目和全年重点。

#### 成果产出趋势图

使用分组柱状图，按月展示正式成果、阶段性进展和重要问题解决。点击柱体查看具体成果。

#### 工作量分类结构图

使用横向堆叠条形图，展示工作类型、产品和子任务的工作当量构成。

#### 能力投入气泡图

采用时间型气泡矩阵：

- 横轴为月份或季度。
- 纵轴为能力名称。
- 气泡面积表示该能力分摊到的工作当量。
- 气泡颜色深浅表示分摊工时。
- 特殊描边表示该周期形成了成果或成长里程碑。
- 悬停显示工时、工作当量、投入占比、成果数量和主要事项。
- 点击查看关联项目、工作记录、成果、重要问题和里程碑。

气泡大小必须按面积映射数值，不按半径映射。能力过多时默认展示重点能力，并支持搜索和筛选。

#### 能力证据图

使用横向条形图比较每项能力关联的工作投入、成果、重要问题和里程碑。每项能力占一行，工作当量与各类证据分别使用独立条带和量纲，不把工作当量与成果数量相加或共用刻度。它负责解释证据，能力投入气泡图负责展示时间变化。

#### 成长里程碑时间轴

按时间展示首次独立完成、角色变化、关键成果和重要突破，用于构建职业成长叙事。

#### 周期对比图

使用并列柱状图对比本周与上周、本月与上月或各季度的投入和产出变化。

#### 工作节奏日历

使用年度日历热力图展示每天的工时或工作当量，用于识别高负荷周期、连续性和长期空档。

#### 投入与成果对应表

默认使用分析表而不是散点图。按项目并列展示投入、成果、问题解决、角色和价值说明，避免把不同成果强行换算为统一分数。

### 12.3 图表交互规则

- 悬停显示准确数值和口径。
- 点击进入来源或形成明确筛选。
- 同一对象跨页面保持颜色一致。
- 单张图表主要数据色控制在五至六种；更多系列通过筛选查看。
- 不展示空坐标轴，数据不足时说明原因和补充方式。
- 图表下方可以展示简短数据结论，但自动结论只描述事实变化，不替用户判断价值。
- 禁止雷达图、排行榜、综合绩效分、装饰性仪表盘和大面积饼图。

## 13. 配置与数据

### 13.1 工作当量标准库

- 按年度或版本管理业务分类、工作类型、产品、子任务、计量单位和折算系数。
- 支持搜索、新增、编辑、排序、启用和停用。
- 支持从 Excel 导入当年初始标准，并在写入前预览新增、重复、冲突和无法识别项。
- 多条同优先级规则同时匹配时提示冲突，不静默选择。
- 无匹配规则时允许人工填写，并标明人工来源。
- 新版本只影响新记录，不自动重算历史数据。
- 不配置传统业务统一折扣。

### 13.2 基础分类

管理业务分类、工作类型、产品、子任务、能力、成果类型、项目状态和计量单位。

已被历史记录使用的选项不能直接删除，只能停用或合并。停用项不再出现在新记录默认选项中，但历史记录保持正常显示和筛选。

### 13.3 数据维护

- 检查未匹配标准、人工系数占比偏高、缺少必要关联、重复分类、疑似重复项目、长期未备份和标准冲突。
- 检查只提醒，不自动修改数据。
- 删除项目、成果或分类前显示受影响对象。
- 支持完整数据备份、校验、恢复和年度归档。
- 支持年度数据及工作当量明细导出为 Excel。

“不保存附件和外部链接”限制工作内容存储，不影响标准导入、完整备份和报告导出。

## 14. 响应式设计

### 14.1 移动端导航

移动端使用顶部栏和抽屉导航，不使用容纳七个模块的底部导航。所有触控目标不小于 `44px`。

### 14.2 移动端重点任务

移动端优先支持日报录入、今日记录查看、项目进展补充和周报检查。月报和年报以阅读和轻量修改为主，完整编辑优先在桌面端完成。

### 14.3 内容适配

- 双栏表单转为单栏。
- 宽表格转为记录列表。
- 标签页支持横向滚动，但当前项始终可见。
- 气泡图和热力图支持横向滑动、点击放大和周期切换。
- 详情侧栏转为全屏页面。
- 文本、按钮和数字不得溢出或遮挡相邻内容。

## 15. 打印与导出体验

- 周报、月报和年报使用独立打印版式，不直接打印应用操作界面。
- 打印时隐藏导航、按钮、筛选器、提示和无关交互控件。
- 报告按 A4 页面组织，避免标题孤立和图表被分页截断。
- Excel 用于工作记录明细、项目投入、成果清单和年度工作当量表。
- Word、PDF 或浏览器打印版用于个人总结和正式汇报。
- 导出前明确显示时间范围、筛选范围、报告折算系数和包含内容。

## 16. 可访问性与动效

- 正文、辅助文字、边框和状态满足清晰对比要求。
- 不只用颜色表达错误、成功、状态或图表重点。
- 所有表单、按钮、菜单、标签页和弹窗支持键盘操作。
- 焦点状态必须清晰可见。
- 图标按钮提供悬停说明和可访问名称。
- 图表提供对应的文字摘要或数据表入口。
- 动效只用于页面切换、状态反馈和侧栏展开，并支持系统“减少动态效果”设置。
- 加载状态保持布局稳定，避免用户正在阅读或输入时发生跳动。

## 17. 实施边界

本轮 UI 改造不扩展以下范围：

- 不增加团队协作、成员分工、审批和权限系统。
- 不增加附件或外部链接管理。
- 不增加个人评分、绩效排名或游戏化激励。
- 不增加自动价值判断或自动撰写个人反思。
- 不为了视觉完整一次性实现所有图表；图表按页面功能与数据基础分阶段交付。
- 不在首轮同时维护深色模式。

## 18. 分阶段落地建议

UI 应随业务能力按依赖关系逐步落地：

1. 建立颜色、字体、间距、控件、导航和页面框架。
2. 重构今日工作台和工作台账，验证高频录入与检索体验。
3. 完成项目管理和成果管理页面，打通投入到产出的证据链。
4. 完成成长与目标页面和能力投入气泡图。
5. 重构周报、月报、年报和数据展板。
6. 完成配置、Excel 标准导入、数据检查、打印和导出体验。

本规范是统一设计来源，但不是一次性交付任务。实施计划必须按以上依赖拆分为可独立验证的阶段，再安排多 agent 协作。各阶段必须复用统一组件和设计变量，不允许 agent 在页面内重新定义独立颜色、圆角、阴影、状态和按钮体系。

## 19. UI 验收标准

### 19.1 一致性

- 七个模块使用统一导航、标题区、筛选栏、状态和操作规则。
- 品牌色、语义色和图表色职责清楚。
- 页面不出现大圆角、渐变、装饰性色块和卡片嵌套回潮。

### 19.2 可用性

- 常用日报能够在较少操作内完成，长选项支持搜索和联动。
- 台账、项目、成果和报告可以相互跳转并返回来源。
- 桌面和移动端均不存在文字、按钮、表格和图表互相遮挡。
- 保存、删除、导入、恢复和冲突状态都有明确反馈。

### 19.3 数据表达

- 工时、原始工作当量、能力分摊和报告折算口径不混淆。
- 多能力、多成果关系不会在界面汇总中重复增加原始工作量。
- 每张图表都能回答明确问题，并可查看来源。
- 能力投入使用时间型气泡图，气泡面积、颜色和特殊描边的含义符合本规范。
- 周报、月报和年报的默认图表与各自周期任务一致。

### 19.4 报告质量

- 报告数据分析和手工复盘内容职责分离。
- 报告预览具有稳定阅读宽度和清晰层级。
- A4 打印版不包含应用导航和操作控件，图表不被无意义截断。
- 年度工作当量 Excel 与个人总结型报告分别满足数据核对和正式汇报需要。

## 20. 已确认的关键决策

- 采用“冷静专业工作台 + 汇报级阅读体验”。
- 使用深青绿色作为品牌色，配合中性灰和独立语义色。
- 采用 `6px` 至 `8px` 圆角、少阴影、无渐变的视觉语言。
- 桌面使用 `216px` 固定分组侧边栏，移动端使用顶部栏和抽屉导航。
- 高频操作页面紧凑，报告页面提高阅读舒适度。
- 日报草稿手动保存，不采用自动保存。
- 项目和成果默认使用紧凑列表，不采用默认看板。
- 项目进度和目标进度必须有状态或证据，不采用任意百分比。
- 能力投入采用时间型气泡图，能力证据采用横向对比图。
- 不以图表数量为目标，不采用雷达图、排行榜和综合个人评分。
- 所有重要统计和图表支持追溯来源。
- 不自动对传统业务工作当量打折。
- 初期不提供深色模式。
